**** Circuit file layout (confirmed 2026-07-02)
:PROPERTIES:
:Why: Avoid duplication, keep surface minimal.
:END:

- lib/niwi/src/circuits/secp256k1_circuit.h — reusable gate-based point ops
  Adapted from Longfellow's circuits/ecdsa/verify_circuit.h:
  - addE, doubleE, is_on_curve, point_equality
  - Scalar multiplication: repeated squaring + 3-bit window table
    (matching the ECDSA triple-scalar pattern, but parameterized for
    double-scalar s·G == R + e·P for BIP-340)
  - x-only lift constraint: even y, x < p, on-curve
  - Range checks (vlt against n), nonzero checks (witness inverse)

- lib/niwi/src/circuits/bip340_circuit.h — BIP-340 verification circuit
  verify_bip340(pk_x, R_x, s, msg[32]):
  - lift both x-only points with even-y constraint
  - e = int(tagged_hash("BIP0340/challenge", R_x || pk_x || msg)) mod n
    computed INSIDE the circuit (tagged hash uses FlatSHA256; messages
    are witness data, the verifier cannot supply e as trusted public input)
  - s·G == R + e·P via double-scalar mult from secp256k1_circuit.h
  - Range: R_x < p, s < n, e < n

- lib/niwi/src/circuits/bip340_witness.* — native witness generation
  fill_witness() producing pre[8], bi[256], int_xyz[255] for the
  scalar-mul loop, plus SHA witness data for tagged_hash

- lib/niwi/src/circuits/rpbsch_circuit.h — RPBSch relation
  - Branch 1 (honest): R' = R + α·G + β·X, even-y R', c = Hq(R'_x, X_x, m) + β mod n,
    C opening (inline Pedersen: C == m·G + r·H)
  - Branch 2 (trapdoor): νᵤ ≠ νᵤ', S opening, two BIP-340 verifications
    under X' on (νₛ, νᵤ) and (νₛ, νᵤ')
  - OR composition with private boolean selector 0=branch1, 1=branch2
    Constraints for inactive branch are gated (not computed-then-chosen)
    Selector asserted boolean
